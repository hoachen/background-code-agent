# Background Code Agent Simulator

This project is a TypeScript and Node.js application that simulates a background code agent, similar to some functionalities of AI coding assistants like Jules. It can receive mock code modification tasks, simulate processing them in the background, and generate results including simulated code diffs and logs.

## Features

-   **Task Submission**: Accepts tasks with repository URL, branch, description, and optional setup scripts.
-   **Background Simulation**: Uses `async/await` and `setTimeout` to simulate long-running background processes.
-   **Progress Logging**: Outputs simulated progress messages during task execution.
-   **Code Modification Simulation**:
    -   Simulates fixing bugs based on task descriptions (e.g., date formatting).
    -   Simulates adding features (e.g., event handlers).
    -   Simulates adding tests.
    -   Operates on an in-memory mock file system.
-   **Result Generation**: Produces a task result object including:
    -   Task ID
    -   Status (completed, failed)
    -   Summary
    -   Simulated code diff
    -   Execution logs
    -   Simulated Pull Request URL
-   **Error Handling**: Simulates task failures due to:
    -   Setup script errors
    -   Code conflicts
    -   Test failures

## Project Structure

-   `src/`: Contains the TypeScript source code.
    -   `types.ts`: Defines core interfaces and types for tasks and results.
    -   `agent.ts`: Contains the `BackgroundCodeAgent` class with the core logic.
    -   `main.ts`: The main executable script that demonstrates the agent's functionality by submitting various sample tasks.
-   `dist/`: Contains the compiled JavaScript code (generated by `tsc`).
-   `package.json`: Project metadata and dependencies.
-   `tsconfig.json`: TypeScript compiler configuration.
-   `.gitignore`: Specifies intentionally untracked files that Git should ignore.

## Prerequisites

-   [Node.js](https://nodejs.org/) (version 14.x or later recommended)
-   [npm](https://www.npmjs.com/) (usually comes with Node.js)

## Installation

1.  Clone the repository (or ensure all files are present).
2.  Navigate to the project directory:
    ```bash
    cd path/to/background-code-agent
    ```
3.  Install the dependencies:
    ```bash
    npm install
    ```

## Running the Simulator

To build the TypeScript code and then run the main simulation script:

```bash
npm start
```

This will:
1.  Compile the TypeScript files from `src/` to JavaScript in `dist/`.
2.  Execute `dist/main.js`, which will submit several predefined tasks to the agent.
3.  The agent will simulate processing these tasks, and you will see log outputs in the console.
4.  After a simulated delay, the final status and details of each task will be printed.

## How it Works

The `BackgroundCodeAgent` class maintains an in-memory representation of a file system (`mockFileSystem`). When a task is submitted:
1.  A unique ID is assigned.
2.  The task is added to a queue (a `Map` in this simulation).
3.  The `processTask` method is invoked asynchronously. This method simulates various stages of a code modification pipeline:
    -   Cloning a repository.
    -   Running setup scripts.
    -   Analyzing code.
    -   Generating a modification plan (based on keywords in the task description).
    -   Applying modifications to the in-memory files.
    -   Generating a textual diff of the changes.
    -   Running tests.
4.  Each step involves simulated delays using `setTimeout` to mimic real-world processing times.
5.  The final result, including logs and status, is stored and can be retrieved using the task ID.
